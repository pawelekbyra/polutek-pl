- access based on `User.isPatron`,
- access based on Clerk metadata,
- access based on Subscription,
- email unsubscribe revoking patron access,

- AI mega-refactor,
- runtime + schema + roadmap in one PR,
- schema migration hidden inside feature ticket,
- package-lock change hidden inside docs/runtime ticket,
- multi-lane cleanup without ticket split,
- target architecture presented as current implementation.
